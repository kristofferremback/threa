Sidebar Organization Exploration

How to handle different scales of streams without user-defined labels

Problem Space

Four scenarios that need different organizational approaches

Scenario 1: No Labels, High Volume
30+ scratchpads, 30+ channels, 30+ DMs. No organization yet.
  • 90+ items in flat list = unusable
  • Can't find anything without search
  • No way to see what's important NOW
  • Scrolling fatigue
Scenario 2: Few Labels, Sparse
3-4 labels with 2-5 streams each. Light usage.
  • Current design works fine
  • But labels might feel forced
  • Empty sections look sparse
Scenario 3: Dense Labels
A few labels but 20+ streams per label. Power user.
  • Sections become too long
  • Need nested scrolling or collapse
  • Color strip loses effectiveness
  • Need density controls
Scenario 4: Many Heavy Labels
10+ labels with 10+ streams each. Organization sprawl.
  • Over-organization problem
  • Can't see overview
  • Needs filter/search to be useful
  • Quick switcher becomes essential

Organizational Strategies

Different approaches to surface important information at scale

Smart Auto-Sections
Recommended
Automatically organize by activity patterns and user behavior, without requiring explicit organization.

Pros

  • Zero setup required
  • Surfaces important items
  • Works before labels exist
  • Adapts to usage patterns

Cons

  • Less predictable
  • May hide wanted items
  • Needs good algorithms
Type-Based Sections
Group by stream type (Scratchpads / Channels / DMs) with collapsible sections.

Pros

  • Predictable structure
  • Easy to understand
  • Natural categories
  • Good for inventory view

Cons

  • Doesn't surface priority
  • Still need to scan all
  • Type may not matter
View Modes + Filters
Multiple view modes (Recent, Type, Alphabetical) with filter controls to show different subsets.

Pros

  • Flexible for power users
  • Can customize view
  • Works at any scale
  • Discoverable

Cons

  • More UI complexity
  • Requires user action
  • May be overwhelming
Density Modes
Toggle between compact (names only) and comfortable (with previews) to fit more items in viewport.

Pros

  • Fits more on screen
  • User preference
  • Quick toggle
  • Reduces scrolling

Cons

  • Loses context in compact
  • Doesn't solve organization
  • Just a display tweak

Implementation Details

How the smart sections algorithm works

‚ö° Important Section
Auto-populated based on signals that demand attention:
  • Unread mentions (@you)
  • Active AI conversations (last 1 hour)
  • Unread messages in pinned streams
  • Recent threads you participated in
Max 10 items. Sorted by urgency score.
üïê Recent Section
Streams with activity in the last 7 days:
  • Any stream you've sent messages in
  • Any stream with new messages (not in Important)
  • Sorted by last activity timestamp
  • Excludes archived/muted streams
Max 15 items. Auto-expires after 7 days.
üìå Pinned Section
User-controlled, always visible:
  • Manually pinned by user
  • Persists regardless of activity
  • Drag to reorder
  • Right-click to unpin
No limit. User decides.
üìÇ Everything Else
All other streams, collapsed by default:
  • Streams not in above sections
  • Click to expand (shows all, scrollable)
  • Use search to find specific items
  • Can switch to "All" view for inventory
Prevents overwhelming the sidebar.
